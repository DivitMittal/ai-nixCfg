## Overview

Custom packages directory. All packages are re-exported as a single attrset from `pkgs/default.nix`.

## Structure

| Path | Purpose |
|------|---------|
| `pkgs/custom/gowa` | Custom `gowa` package |
| `pkgs/darwin/` | macOS-specific packages (e.g. Perplexity-bin) |
| `pkgs/_sources/` | Generated by nvfetcher — never edit manually |

## nvfetcher Sources

`pkgs/_sources/` is fully generated. To update sources, use the `pkgs-update` command available in the devshell:

```bash
nix develop
pkgs-update
```

Never edit files under `pkgs/_sources/` by hand; changes will be overwritten on the next `pkgs-update` run.

## Conventions

- New packages go under `pkgs/custom/` (cross-platform) or `pkgs/darwin/` (macOS-only).
- `pkgs/default.nix` must be updated to re-export any new package added.
- Darwin-only packages should guard against non-Darwin builds where appropriate.
